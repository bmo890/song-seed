# songnook-word-he

The Hebrew Word Finder's entire backend. Two upstreams behind one worker:

- **Meaning modes** (synonyms, similar, opposites, related, describing, kinds,
  parts, homophones, definition) → generated by Claude Haiku (**paid**). No
  Hebrew API exists for these.
- **Sound modes** (rhymes + nikud disambiguation) → proxied from Dicta's free
  **Charuzit** service. Proxied (not called from the app) so results are cached
  and Dicta's unversioned endpoint is swappable with a redeploy.

```
app  →  this worker ──┬─ /he/words          →  Claude API (Haiku; paid)
                      ├─ /he/vocalizations  →  Charuzit (free)
                      └─ /he/rhymes         →  Charuzit (free)
```

## Contract

```
POST /he/words          { word, mode, themes?: string[] }
  word-list modes  →  200 { words: [{ word, syllables, tag? }] }
  mode "definition" → 200 { definition: { pos, gloss, root? } }
POST /he/vocalizations  { word }
  → 200 { options: ["שָׁמַיִם", …] }   nikud readings, most common first
POST /he/rhymes         { vocal }   one pointed reading
  → 200 { words: [{ word, vocal }] }   word = unvocalized for insertion
All: 400 bad input (never billed) · 429 upstream rate limit · 502 upstream failure
```

Modes: `synonyms` `similar` `opposites` `related` `describing` `kinds` `parts`
`homophones` `definition`. The app maps its Datamuse-era mode ids onto these in
`src/domain/wordTools.ts`.

## Deploy

```sh
npm install
npx wrangler deploy
npx wrangler secret put ANTHROPIC_API_KEY   # Console key from platform.claude.com
```

Set a **monthly spend limit** on the key's workspace in the Anthropic Console.
Local dev: put `ANTHROPIC_API_KEY=sk-ant-...` in `.dev.vars` (gitignored), then
`npx wrangler dev`.

> **KV note:** `wrangler kv key get/list/put` default to **local** storage.
> To inspect what the deployed worker actually wrote, pass `--remote`.

## Caching & cost

Every route has two cache tiers: the edge cache (fast, per-datacenter,
evictable) in front of **KV** (`WORDHE_KV`, durable + global). A given lookup is
computed at most once, ever, worldwide — per `PROMPT_VERSION` (meaning) /
`CHARUZIT_VERSION` (sound). Charuzit routes cost nothing. A Haiku meaning lookup
is ~400 input + ~700 output tokens ($1/$5 per MTok) ≈ **$0.002 uncached, then
free forever**. `max_tokens` caps a single request; the Console spend limit caps
the month. Rhymes (the default mode) never hit the LLM at all.

## Prompt tuning / eval

The prompts in `worker.js` are the product surface — tune them against a fixed
word list before wiring the app. Suggested eval set (run each through every
mode and eyeball):

- Everyday nouns: אהבה, בית, דרך, לילה, ים
- Verbs: ללכת, לשבור, להתגעגע
- Adjectives (agreement check — expect matching gender/number): יפה, שבורה, ישנים
- Register extremes: נשיקה (poetic field), באסה (slang), חסד (biblical flavor)
- Homophone traps: קרא / קרה / כרה, עלה / אלה
- Loanwords / geresh: ג'ינס, צ'יק

What to look for: no invented words, no nikud in output, gender-agreeing
adjectives in `describing`, tags only when informative, sensible ordering.
